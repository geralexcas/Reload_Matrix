## 1. Error 500 al crear socio sin DV

**Problema:** `TypeError: object of type 'NoneType' has no len()` en `_validate_nit_dv` cuando `dv` es `null`.

**Causa:** El frontend envía `dv: form.value.dv || null`. Para CC/CE/Pasaporte no hay campo DV, queda `""` → `null`. El backend no manejaba `None`.

**Fix:** `backend/app/services/partner_service.py:80,120`
```python
# Antes
if not self._validate_nit_dv(partner.nit, partner.dv):
# Después
if partner.dv and not self._validate_nit_dv(partner.nit, partner.dv):
```

**Frontend:** `frontend/src/views/Partners/PartnerFormView.vue:172-175`
```javascript
if (form.value.document_type === 'NIT' && !form.value.dv) {
  alert('Para NIT debe ingresar el Dígito de Verificación (DV)')
  return
}
```

## 2. Treasury sync roto al publicar asientos contables

**Problema:** Al publicar un asiento contable con crédito a 1110, no se actualizaba el saldo de la cuenta bancaria.

**Causa:** El commit `f7904f1` cambió prefix-matching por exact-matching en el fallback de treasury sync.

| Elemento | Detalle |
|----------|---------|
| Banco vinculado a | `111011 - Bancos - Cuenta de ahorros` |
| Asiento contable usó | `1110 - Bancos (Transferencias)` |
| Match exacto (`==`) | `"1110" == "111011"` → **FALSO** |
| Match prefijo | `"111011".startswith("1110")` → **VERDADERO** |

**Fix:** `backend/app/services/accounting_service.py:385,441`
```python
# Antes
if line_code == c_code:
# Después
if line_code.startswith(c_code) or c_code.startswith(line_code):
```

**Rebuild:**
```bash
docker compose up -d --build backend
```

## 3. Libro de Compras - Base IVA incorrecta

**Problema:** Items con `tax_rate=19` pero `tax_amount=0` aparecían como Base IVA 19% mostrando el costo total.

**Causa:** `_get_invoice_tax_breakdown` clasificaba por `tax_rate` sin verificar si realmente hay `tax_amount > 0`.

**Fix:** `backend/app/services/accounting_service.py:762-768`
```python
# Antes
if self._is_approx_equal(tax_rate, Decimal("0.19")):
    base_iva_19 += line_base
    iva_19 += item_tax_amount
elif self._is_approx_equal(tax_rate, Decimal("0.05")):
    base_iva_5 += line_base
    iva_5 += item_tax_amount
else:
    base_no_iva += item.line_total

# Después
if item_tax_amount > 0 and self._is_approx_equal(tax_rate, Decimal("0.19")):
    base_iva_19 += line_base
    iva_19 += item_tax_amount
elif item_tax_amount > 0 and self._is_approx_equal(tax_rate, Decimal("0.05")):
    base_iva_5 += line_base
    iva_5 += item_tax_amount
else:
    base_no_iva += item.line_total
```

## 4. Comandos útiles para diagnóstico en producción

### Ver cuentas PUC 1110
```bash
docker compose exec db psql -U user -d business_db -c "
SELECT id, code, name, is_active FROM chart_of_accounts
WHERE code LIKE '1110%' AND company_id = 1 ORDER BY code;
"
```

### Ver vinculación banco ↔ PUC
```bash
docker compose exec db psql -U user -d business_db -c "
SELECT ba.id, ba.name, ba.linked_account_id, coa.code, coa.name as coa_name
FROM bank_accounts ba
LEFT JOIN chart_of_accounts coa ON coa.id = ba.linked_account_id
WHERE ba.is_active = true;
"
```

### Ver ítems de compra por número de factura
```bash
docker compose exec db psql -U user -d business_db -c "
SELECT pi.id, pi.description, pi.line_total, pi.tax_rate, pi.tax_amount, p.purchase_number
FROM purchase_items pi
JOIN purchases p ON p.id = pi.purchase_id
WHERE p.purchase_number IN ('FECO 66687', 'ATPE 100488', 'ATPE 100508')
ORDER BY p.purchase_number, pi.id;
"
```

### Corregir tax_rate donde tax_amount = 0 (si aplica)
```bash
docker compose exec db psql -U user -d business_db -c "
UPDATE purchase_items
SET tax_rate = 0
WHERE id IN (
    SELECT pi.id FROM purchase_items pi
    JOIN purchases p ON p.id = pi.purchase_id
    WHERE p.purchase_number IN ('FECO 66687', 'ATPE 100488', 'ATPE 100508')
    AND (pi.tax_rate IS NOT NULL AND pi.tax_rate > 0)
    AND (pi.tax_amount IS NULL OR pi.tax_amount = 0)
);
"
```
